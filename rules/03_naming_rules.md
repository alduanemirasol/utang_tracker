# Naming Rules

- Use snake_case for database tables and columns.
- Use PascalCase for classes.
- Use camelCase for variables and methods.
- Use UPPER_SNAKE_CASE for enum values.
- Suffix providers with `Provider`.
- Suffix repositories with `Repository`.
- Suffix repository implementations with `RepositoryImpl`.
- Suffix use cases with `UseCase`.
- Suffix DTOs with `Dto`.
- Suffix database models with `Model`.
- One class per file.
- File names use snake_case.
